SQL Injection (SQLi) - introduction and SQLi in login fields

After learning in previous articles how to install the Linux terminal on PC and Android, the main commands in this OS, as well as various legal aspects of hacking, in today's article it is time to move on to actual ethical hacking.

Note: Hacking a device or website without the owner's consent is illegal. Hacking techniques can only be used legally in environments specifically designed for testing such skills, on your own devices, or in a setting where you have the owner's permission.

The first technique we will talk about is called SQL Injection. By exploiting this type of vulnerability, an attacker can gain unauthorized access to a website's database and can execute command through the input fields of the site that interact with the database. In this multi-part lesson, you will learn what SQL Injection is, how to legally exploit this vulnerability, and how to protect yourself against this type of attack.


How does SQL Injection works

First of all, we need to understand that website databases are created using a programming language called SQL (Structured Query Language). As you probably already know, each programming language has its specific purpose:

HTML is used for creating websites

CSS is used for styling website content

JavaScript is used for creating scripts on websites

PHP is used for connecting websites to servers, etc.

Therefore, SQL is used for creating databases and managing the connection between a website and its database.

Let's imagine that we have a website that has the following table called "users" in the database, where the login information of the website's users is stored.

ID Username Password City
1 john_doe password123 New York
2 jane_smith qwerty456 Los Angeles
3 mike_jones letmein789 Chicago
4 anna_white securepass New York
5 chris_brown admin123 Los Angeles
